Project 1

Latest News Summarizer & Question Generator

This project implements a RAG pipeline based on ElasticSearch and LangChain, connecting to the latest documents and providing question generation and accurate answers grounded in the given context.


This project uses a Retrieval-Augmented Generation (RAG) architecture: indexing the latest documents in ElasticSearch, retrieving related context with LangChain retrievers, and constraining the LLM to respond only based on that context.

Tech Stack

  • RAG: BM25 + (optional) vector search, source document citation
  • LangChain: Text Splitter (RecursiveCharacterTextSplitter), MultiQuery Retriever, ChatPromptTemplate
  • Flask (Python): REST/SSE API, CORS, (deployment) gunicorn/waitress
  • Elastic Cloud (Elasticsearch): single index, sparse/dense field mapping
  • Selenium: headless crawling, cleaning/deduplication

Structure

Data Crawling & JSON Mapping

  • Crawl latest news/documents with Selenium → extract body/title/URL/published date
  • Remove HTML tags, normalize spaces, and map to JSON schema

Text Splitting with LangChain

  • RecursiveCharacterTextSplitter (e.g., chunk_size=800, chunk_overlap=120)
  • Preserve metadata: source, url, published_at, etc.

Elasticsearch Indexing

  • Index: rag-docs
  • Fields: content(text), source(keyword), url(keyword), published_at(date), embedding(dense_vector)
  • Upsert to prevent duplication, reindex on update date for quick refresh of new docs

GPT Model Integration & Prompts/Questions

  • Model: gpt-4o-mini
  • System prompt: “Answer only within the given context; if no evidence, say ‘I don’t know.’”
  • Example questions: “Summarize the text”, “Which university is the author affiliated with?”, “Extract only the key numbers”

Flask

  • Python backend API connecting to frontend (e.g., POST /api/ask, GET /api/stream for SSE)
